# 桌面智能工具需求文档

## 1. 项目概述

### 1.1 项目背景
随着日常工作和学习中重复性任务的增多，用户需要一款能够自动化执行简单任务、支持自定义技能扩展的桌面智能工具，提高工作效率。

### 1.2 项目目标
开发一个跨平台的桌面智能工具，具备基础功能执行能力，并支持通过自定义技能（Skills）扩展来完成特定领域的工作任务。同时，支持对接飞书、企业微信等即时通讯平台，通过对话方式自然地执行任务，实现"聊天即操作"的工作体验。

### 1.3 目标用户
- 需要自动化重复性日常任务的用户
- 开发者和高级用户，希望自定义工具功能
- 需要集成多个工具和服务的办公人员

## 4. 技术架构建议

### 4.1 技术栈选项

#### 选项 A：Electron + Node.js
- **优势**：跨平台、生态丰富、开发效率高
- **劣势**：资源占用相对较大
- **适用场景**：快速开发、需要丰富的前端能力

#### 选项 B：Tauri + Rust
- **优势**：轻量、性能好、安全性高
- **劣势**：学习曲线较陡
- **适用场景**：注重性能和安全性

#### 选项 C：Python + PyQt/PySide
- **优势**：Python 生态强大、技能开发门槛低
- **劣势**：打包分发相对复杂
- **适用场景**：以 Python 技能为主

### 4.3 技能定义格式示例

```yaml
# skill.yaml
name: "文本提取器"
version: "1.0.0"
description: "从文件中提取特定格式的文本"
author: "Your Name"
tags: ["文本处理", "提取"]
main: "index.py"

# 聊天相关配置
dialog:
  # 意图配置
  intents:
    - name: "extract_text"
      examples:
        - "提取文件中的文本"
        - "帮我从{file_path}提取{pattern}内容"
        - "搜索文件内容"
      triggers:
        - keyword: "提取"
        - regex: "提取.*文本"
        - nlu: "text.extract"

  # 参数提取配置
  slots:
    - name: "file_path"
      type: "string"
      required: true
      description: "要处理的文件路径"
      prompts:
        - "请告诉我文件路径"
        - "需要处理哪个文件？"

    - name: "pattern"
      type: "string"
      required: true
      description: "正则表达式模式"
      prompts:
        - "请提供要匹配的模式"
        - "需要匹配什么内容？"

  # 响应模板
  response:
    success:
      type: "card"
      template: |
        ## 提取结果
        文件: {{file_path}}
        匹配数: {{count}}

        {{#results}}
        - {{.}}
        {{/results}}

    error:
      type: "text"
      template: "提取失败: {{error}}"

    progress:
      type: "text"
      template: "正在处理中... {{progress}}%"

parameters:
  - name: "file_path"
    type: "string"
    required: true
    description: "要处理的文件路径"
  - name: "pattern"
    type: "string"
    required: true
    description: "正则表达式模式"

outputs:
  - name: "results"
    type: "array"
    description: "匹配结果列表"
  - name: "count"
    type: "number"
    description: "匹配数量"
```

## 6.2 P1 高优先级风险详解

### 6.2.1 技能安全风险

**风险等级**: P1（高优先级）

**风险描述**
- 恶意技能可能删除系统文件、窃取用户数据
- 技能代码中可能包含后门或恶意代码
- 第三方技能来源不可信，存在供应链攻击风险
- 技能执行可能消耗过多系统资源

**最佳方案推荐** ⭐⭐⭐⭐⭐

**首选方案：权限声明与控制**
- **推荐理由**:
  - 实施成本中等，技术难度适中
  - 用户体验好，透明度高
  - 可与沙箱机制结合，效果倍增
  - 符合现代应用安全最佳实践
- **预计工期**: 4周
- **实施优先级**: 最高（必须实施）

**次选方案：进程级沙箱隔离**（中期目标）
- **推荐理由**:
  - 安全性最高，能够彻底隔离
  - 实施成本高，跨平台复杂
  - 长期维护成本较高
- **预计工期**: 4周
- **建议**: 作为中期目标，优先实现权限控制，后续逐步完善沙箱

**技术方案**

#### 方案一：进程级沙箱隔离

**技术选型**
- **Linux**: 使用 `seccomp`、`cgroups`、`namespaces` 实现资源隔离
- **macOS**: 使用 `sandbox-exec` 实现沙箱
- **Windows**: 使用 Windows Job Objects 和 AppContainer

**实施步骤**
1. 设计沙箱配置格式
2. 实现资源配额限制（CPU、内存、磁盘 I/O）
3. 实现文件系统虚拟化，限制访问范围
4. 实现网络访问白名单
5. 编写沙箱测试用例

```python
import hmac
import hashlib

def verify_feishu_signature(timestamp, nonce, body, signature, encrypt_key):
    # 拼接签名数据
    sign_str = timestamp + nonce + encrypt_key + body
    # 计算 HMAC-SHA256
    expected = hmac.new(
        encrypt_key.encode(),
        sign_str.encode(),
        hashlib.sha256
    ).hexdigest()
    # 比较签名
    return hmac.compare_digest(expected, signature)
```

```python
class ReplayProtection:
    def __init__(self, redis_client):
        self.redis = redis_client
        self.tolerance = timedelta(minutes=5)  # 5分钟时间窗口

    def check(self, message_id: str, timestamp: int) -> bool:

        if abs(now - timestamp) > self.tolerance.total_seconds():
            return False

        # 记录已处理消息

        return True
```

```python
from typing import Dict, List, Optional
import re

class SlotFiller:
    def __init__(self):
        self.slots = {
            "file_path": r'[~/]?[\w/\\.-]+\.\w+',
            "pattern": r'[\'"](.*?)[\'"]',
            "number": r'\b\d+\b',

        for slot_name, pattern in self.slots.items():
            match = re.search(pattern, text)
            if match:
                filled[slot_name] = match.group(1) if match.groups() else match.group(0)

        # 检查必填槽位
        missing = [s for s in required_slots if s not in filled]
        if missing:
            raise MissingSlotsError(missing)

        return filled

    def ask_for_slots(self, missing_slots: List[str]) -> str:
        """生成询问消息"""
        prompts = {
            "file_path": "请提供文件路径",
            "pattern": "请提供匹配模式",
            "number": "请提供数字",
        }
        return "、".join([prompts.get(s, f"请提供{s}") for s in missing_slots])
```

```python
import asyncio
from concurrent.futures import ProcessPoolExecutor
from typing import Callable, Any
import uuid

class TaskQueue:
    def __init__(self, max_workers: int = 4):
        self.executor = ProcessPoolExecutor(max_workers=max_workers)
        self.tasks = {}

    async def submit(self, func: Callable, *args, **kwargs) -> str:
        """提交任务"""
        task_id = str(uuid.uuid4())
        loop = asyncio.get_event_loop()

        def task_wrapper():
            try:
                result = func(*args, **kwargs)
                self.tasks[task_id] = {"status": "completed", "result": result}
                return result
            except Exception as e:
                self.tasks[task_id] = {"status": "failed", "error": str(e)}
                raise

        future = loop.run_in_executor(self.executor, task_wrapper)
        self.tasks[task_id] = {"status": "running", "future": future}
        return task_id

    async def get_result(self, task_id: str) -> Any:
        """获取任务结果"""
        task = self.tasks.get(task_id)
        if not task:
            raise TaskNotFoundError(task_id)

        if task["status"] == "completed":
            return task["result"]
        elif task["status"] == "failed":
            raise TaskExecutionError(task["error"])
        elif task["status"] == "running":
            return await task["future"]
```

```python
class ProgressTracker:
    def __init__(self, platform_adapter):
        self.adapter = platform_adapter
        self.progress = {}

    def start(self, task_id: str, total: int, chat_id: str):
        """开始进度跟踪"""

            "current": 0,
            "total": total,
            "chat_id": chat_id,
            "start_time": time.time()

    def update(self, task_id: str, current: int, message: str = None):

        task = self.progress[task_id]
        task["current"] = current
        percent = (current / task["total"]) * 100

        # 每完成 10% 推送一次
        if current == 0 or current == task["total"] or current % (task["total"] // 10) == 0:
            self.adapter.send_message(
                task["chat_id"],
                f"进度: {percent:.0f}% ({current}/{task['total']}){': ' + message if message else ''}"
            )

    def complete(self, task_id: str, result: str = None):
        """完成进度跟踪"""
        if task_id in self.progress:
            task = self.progress[task_id]
            elapsed = time.time() - task["start_time"]
            self.adapter.send_message(
                task["chat_id"],
                f"✅ 完成! 耗时: {elapsed:.1f}秒{' - ' + result if result else ''}"
            )
            del self.progress[task_id]
```

#### 方案二：配置备份与恢复

**技术实现**
```python
import shutil
from pathlib import Path
from datetime import datetime

class ConfigBackup:
    def __init__(self, config_dir: Path, backup_dir: Path, max_backups: int = 10):
        self.config_dir = config_dir
        self.backup_dir = backup_dir
        self.max_backups = max_backups

    def backup(self) -> str:
        """备份配置"""
        timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
        backup_path = self.backup_dir / f"backup_{timestamp}"
        shutil.copytree(self.config_dir, backup_path)

        # 清理旧备份
        self._cleanup_old_backups()

        return str(backup_path)

    def restore(self, backup_id: str) -> bool:
        """恢复配置"""
        backup_path = self.backup_dir / f"backup_{backup_id}"
        if not backup_path.exists():
            return False

        # 备份当前配置
        self.backup()

        # 恢复
        shutil.rmtree(self.config_dir)
        shutil.copytree(backup_path, self.config_dir)
        return True

    def _cleanup_old_backups(self):
        """清理旧备份"""
        backups = sorted(self.backup_dir.glob("backup_*"), reverse=True)
        for old_backup in backups[self.max_backups:]:
            shutil.rmtree(old_backup)
```

**实施步骤**

2. 实现恢复功能
3. 实现自动清理
4. 提供恢复 UI

**验收标准**

### 8.2 术语表
- **技能（Skill）**：用户或第三方开发的可执行功能模块
- **插件（Plugin）**：扩展系统核心功能的模块

- **意图（Intent）**：用户想要执行的操作的语义表示
- **槽位（Slot）**：对话中需要从用户消息中提取的参数
- **会话（Session）**：用户与机器人之间的一系列交互的上下文
- **平台适配器（Platform Adapter）**：对接不同聊天平台的抽象层组件
