## 2. 接口概述

### 2.1 接口分类

```
接口类型
├── 前端-后端接口 (Frontend-Backend API)
│   └── Tauri Command 接口
├── Webhook 接口 (Webhook API)
│   ├── 飞书 Webhook
│   ├── 企业微信 Webhook
│   └── 钉钉 Webhook
├── 管理接口 (Admin API)
│   └── HTTP REST API
└── 技能接口 (Skill API)
    ├── 技能执行接口
    └── 技能回调接口
```

### 2.2 通用规范

#### 请求/响应格式

**统一响应结构**:
```json
{
  "code": 0,                    // 状态码：0 成功，非 0 失败
  "message": "success",         // 消息
  "data": {},                   // 响应数据
  "timestamp": 1703222400       // 时间戳
}
```

**错误响应**:
```json
{
  "code": 1001,
  "message": "参数验证失败",
  "errors": [
    {
      "field": "file_path",
      "message": "文件路径不能为空"
    }
  ],
  "timestamp": 1703222400
}
```

**状态码定义**:

| 状态码 | 说明 |
|--------|------|
| 0 | 成功 |
| 1000-1999 | 通用错误 |
| 1000 | 未知错误 |
| 1001 | 参数验证失败 |
| 1002 | 未授权 |
| 1003 | 资源不存在 |
| 2000-2999 | 技能相关错误 |
| 2000 | 技能执行失败 |
| 2001 | 技能不存在 |
| 2002 | 技能未启用 |
| 3000-3999 | 权限相关错误 |
| 3000 | 权限不足 |
| 3001 | 权限被拒绝 |
| 4000-4999 | 平台相关错误 |
| 4000 | 平台适配器错误 |
| 4001 | Webhook 签名验证失败 |
| 4002 | 消息格式错误 |

## 3. 前端-后端接口 (Tauri Commands)

### 3.1 技能管理接口

#### 3.1.1 获取技能列表

**接口**: `skill:list`

**请求参数**:
```typescript
interface SkillListParams {
  filter?: {
    enabled?: boolean;      // 是否只显示启用的技能
    tag?: string;           // 按标签过滤
    category?: string;      // 按分类过滤
  };
  pagination?: {
    page: number;           // 页码，从 1 开始
    page_size: number;      // 每页数量
  };
  sort?: {
    field: 'name' | 'installed_at' | 'execution_count';
    order: 'asc' | 'desc';
  };
}
```

**响应数据**:
```typescript
interface SkillListResponse {
  skills: Array<{
    id: string;
    name: string;
    version: string;
    description: string;
    author: string;
    tags: string[];
    enabled: boolean;
    installed_at: number;
    execution_count: number;
    icon?: string;
  }>;
  total: number;
  page: number;
  page_size: number;
}
```

**示例**:
```rust
#[tauri::command]
async fn list_skills(
    filter: Option<SkillFilter>,
    pagination: Pagination,
    sort: Option<Sort>,
    state: State<'_, AppState>,
) -> Result<SkillListResponse, AppError> {
    state.skill_service.list(filter, pagination, sort).await
}
```

## 4. Webhook 接口

### 4.1 飞书 Webhook

#### 4.1.1 接收消息

**端点**: `POST /webhook/feishu`

**请求头**:
```
Content-Type: application/json
X-Lark-Request-Timestamp: {timestamp}
X-Lark-Request-Nonce: {nonce}
X-Lark-Signature: {signature}
```

**请求体**:
```json
{
  "encrypt": "{encrypted_data}"
}
```

**响应**:
```json
{
  "code": 0,
  "msg": "success"
}
```

**处理流程**:
```
1. 验证签名
2. 解密消息
3. 解析事件类型
4. 提取消息内容
5. 识别意图
6. 执行操作
7. 发送响应
```

#### 4.1.2 发送消息

**发送文本消息**:
```json
{
  "msg_type": "text",
  "content": {
    "text": "Hello World"
  }
}
```

**发送卡片消息**:
```json
{
  "msg_type": "interactive",
  "card": {
    "config": {
      "wide_screen_mode": true
    },
    "header": {
      "title": {
        "content": "标题",
        "tag": "plain_text"
      }
    },
    "elements": [
      {
        "tag": "div",
        "text": {
          "content": "内容",
          "tag": "lark_md"
        }
      },
      {
        "tag": "action",
        "actions": [
          {
            "tag": "button",
            "text": {
              "content": "确认",
              "tag": "plain_text"
            },
            "type": "primary",
            "value": {
              "action": "confirm"
            }
          }
        ]
      }
    ]
  }
}
```

## 6. 技能接口 (Skill API)

### 6.1 技能执行接口

技能执行时，系统会向技能传递标准输入，并接收标准输出。

#### 6.1.1 输入格式

**命令行参数**:
```bash
skill.py --input /tmp/input.json --output /tmp/output.json
```

**输入文件 (input.json)**:
```json
{
  "skill_id": "file-rename",
  "version": "1.0.0",
  "parameters": {
    "path": "/home/user/pictures",
    "pattern": "*.jpg",
    "template": "Photo_{:03d}.jpg",
    "start_number": 1
  },
  "context": {
    "user_id": "user_123",
    "chat_id": "oc_xxxxx",
    "platform": "feishu",
    "session_id": "feishu:user_123:oc_xxxxx"
  },
  "config": {
    "timeout": 300,
    "dry_run": false
  }
}
```

#### 6.1.2 输出格式

**输出文件 (output.json)**:
```json
{
  "success": true,
  "message": "重命名完成",
  "data": {
    "renamed_files": [
      {
        "from": "/home/user/pictures/IMG_0001.jpg",

  ]

```

```typescript
const unlisten = await listen<SkillProgressEvent>('skill:progress', (event) => {
  console.log('Progress:', event.payload);
});

// 取消订阅
unlisten();
```

#### 7.1.2 事件类型

| 事件名 | 数据类型 | 说明 |
|--------|---------|------|
| `skill:progress` | SkillProgressEvent | 技能执行进度 |
| `skill:completed` | SkillCompletedEvent | 技能执行完成 |
| `skill:failed` | SkillFailedEvent | 技能执行失败 |
| `message:received` | MessageReceivedEvent | 收到新消息 |
| `session:updated` | SessionUpdatedEvent | 会话更新 |
| `config:changed` | ConfigChangedEvent | 配置变更 |
| `notification` | NotificationEvent | 系统通知 |

## 9. 限流与配额

### 9.1 限流策略

| 操作类型 | 限制 | 时间窗口 |
|---------|------|---------|
| 技能执行 | 10 次/分钟 | 60 秒 |
| API 调用 | 100 次/分钟 | 60 秒 |
| 文件上传 | 10 次/小时 | 3600 秒 |
| Webhook 请求 | 60 次/分钟 | 60 秒 |

### 9.2 限流响应

**HTTP 状态码**: `429 Too Many Requests`

**响应体**:
```json
{
  "code": 4300,
  "message": "请求过于频繁",
  "details": {
    "limit": 10,
    "remaining": 0,
    "reset_at": 1703222460
  }
}
```

**响应头**:
```
X-RateLimit-Limit: 10
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1703222460
```
