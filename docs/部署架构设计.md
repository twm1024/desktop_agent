## 2. 部署架构概述

### 2.1 部署模式

本项目为**本地客户端应用**，采用以下部署模式：

```
┌─────────────────────────────────────────────────────────────┐
│                    用户本地环境                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │         Desktop Agent (Tauri 应用)                    │  │
│  │  ┌──────────────┐  ┌──────────────┐                 │  │
│  │  │  主进程      │  │  渲染进程    │                 │  │
│  │  │  (Rust)      │  │  (React)     │                 │  │
│  │  └──────────────┘  └──────────────┘                 │  │
│  │  ┌──────────────────────────────────────────────┐  │  │
│  │  │  内置 Web 服务器 (localhost:8080)            │  │  │
│  │  │  • Webhook 端点                               │  │  │
│  │  │  • 管理 API                                   │  │  │
│  │  │  • WebSocket (可选)                           │  │  │
│  │  └──────────────────────────────────────────────┘  │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │

│  │              本地数据存储                            │  │

│  │  ├── config/         (YAML 配置)                    │  │
│  │  ├── skills/         (技能文件)                     │  │
│  │  ├── logs/           (日志文件)                     │  │
│  │  └── cache/          (缓存文件)                     │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
                           │
                           │ HTTPS / Webhook
                           ▼
┌─────────────────────────────────────────────────────────────┐

│  ┌──────────┐  ┌──────────┐  ┌──────────┐                  │

│  └──────────┘  └──────────┘  └──────────┘                  │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 部署特点

- **本地部署**：无需云端服务器，完全在用户本地运行
- **内网穿透（可选）**：可通过内网穿透服务实现远程访问
- **数据本地化**：所有数据存储在用户本地，不上传云端
- **离线可用**：核心功能不依赖网络，网络仅用于平台 Webhook
- **轻量级**：资源占用低，适合长时间后台运行

## 6. 系统集成

### 6.1 开机自启动

#### Windows
```rust
// 使用 Win32 API 注册开机启动
use winreg::enums::*;
use winreg::RegKey;

pub fn set_auto_start(enable: bool) -> Result<(), WindowsError> {
    let hkcu = RegKey::predef(HKEY_CURRENT_USER);
    let path = hkcu.open_subkey_with_flags(
        r"Software\Microsoft\Windows\CurrentVersion\Run",
        KEY_WRITE,
    )?;

    if enable {
        let exe_path = std::env::current_exe()?;
        path.set_value("DesktopAgent", &exe_path.to_string_lossy())?;
    } else {
        path.delete_value("DesktopAgent")?;
    }

    Ok(())
}
```

```rust
pub fn register_file_association() -> Result<(), WindowsError> {
```

### 7.3 更新实现

```rust
// src/update/updater.rs

use tauri::updater::UpdaterBuilder;

pub struct Updater {
    window: Window,
    config: UpdateConfig,
}

impl Updater {
    pub async fn check_for_updates(&self) -> Result<UpdateInfo, UpdateError> {
        let client = reqwest::Client::new();

        let response = client
            .get(&self.config.check_url)
            .query(&[
                ("version", env!("CARGO_PKG_VERSION")),
                ("platform", std::env::consts::OS),
                ("arch", std::env::consts::ARCH),
                ("channel", if self.config.beta_channel { "beta" } else { "stable" }),
            ])
            .send()
            .await?;

        let info: UpdateInfo = response.json().await?;

        Ok(info)
    }

    pub async fn download_update(&self, info: &UpdateInfo) -> Result<PathBuf, UpdateError> {
        let client = reqwest::Client::new();
        let mut response = client.get(&info.download_url).send().await?;

        let temp_dir = std::env::temp_dir();
        let file_path = temp_dir.join(format!("desktop-agent-{}.{}",
            info.version,
            if cfg!(windows) { "exe" } else { "dmg" }
        ));

        let mut file = File::create(&file_path)?;
        while let Some(chunk) = response.chunk().await? {
            file.write_all(&chunk)?;
        }

        // 验证签名
        self.verify_signature(&file_path, &info.signature)?;

        Ok(file_path)
    }

    pub async fn install_update(&self, package_path: &Path) -> Result<(), UpdateError> {
        // 关闭应用
        self.emit_update_event("installing")?;

        // 调用安装程序
        #[cfg(windows)]
        {
            Command::new(package_path)
                .arg("/SILENT")
                .arg("/NOUI")
                .spawn()?;
        }

        #[cfg(target_os = "macos")]
        {
            Command::new("open")
                .arg(package_path)
                .spawn()?;
        }

        // 退出应用
        std::process::exit(0);
    }

    fn verify_signature(&self, file_path: &Path, signature: &str) -> Result<(), UpdateError> {

    }
}
```

### 10.2 备份实现

```rust
// src/backup/backup.rs

use tokio::fs;

pub struct BackupManager {
    config_dir: PathBuf,
    backup_dir: PathBuf,
    max_backups: usize,
}

impl BackupManager {
    pub async fn create_backup(&self) -> Result<BackupInfo, BackupError> {
        let timestamp = chrono::Local::now().format("%Y%m%d_%H%M%S").to_string();
        let backup_id = format!("backup_{}", timestamp);
        let backup_path = self.backup_dir.join(&backup_id);

        // 创建备份目录
        fs::create_dir_all(&backup_path).await?;

        // 1. 备份配置文件
        self.backup_config(&backup_path).await?;

        // 2. 备份数据库
        self.backup_database(&backup_path).await?;

        // 3. 备份技能清单
        self.backup_skill_manifests(&backup_path).await?;

        // 4. 压缩备份
        if self.config.compression {
            self.compress_backup(&backup_path).await?;
        }

        // 5. 清理旧备份
        self.cleanup_old_backups().await?;

        Ok(BackupInfo {
            id: backup_id,
            path: backup_path,
            timestamp: chrono::Local::now(),
            size: self.calculate_size(&backup_path),
        })
    }

    pub async fn restore_backup(&self, backup_id: &str) -> Result<(), BackupError> {
        let backup_path = self.backup_dir.join(backup_id);

        if !backup_path.exists() {
            return Err(BackupError::NotFound(backup_id.to_string()));
        }

        // 1. 停止服务
        self.stop_services().await?;

        // 2. 创建当前状态备份
        self.create_backup().await?;

        // 3. 解压备份（如果需要）
        let restore_path = if backup_path.extension() == Some("zip".as_ref()) {
            self.extract_backup(&backup_path).await?
        } else {
            backup_path.clone()
        };

        // 4. 恢复配置
        self.restore_config(&restore_path).await?;

        // 5. 恢复数据库
        self.restore_database(&restore_path).await?;

        // 6. 重启服务
        self.restart_services().await?;

        Ok(())
    }
}
```

## 11. 故障恢复

### 11.1 崩溃恢复

```rust
// src/app/recovery.rs

pub struct RecoveryManager {
    crash_log_path: PathBuf,
}

impl RecoveryManager {
    /// 处理应用崩溃
    pub async fn handle_crash(&self) -> Result<(), RecoveryError> {
        // 1. 记录崩溃信息
        self.log_crash().await?;

        // 2. 保存当前状态
        self.save_state().await?;

        // 3. 检查数据库完整性
        self.check_database_integrity().await?;

        // 4. 恢复未完成的任务
        self.recover_tasks().await?;

        Ok(())
    }

    /// 数据库恢复
    pub async fn recover_database(&self) -> Result<(), RecoveryError> {
        let db_path = self.config_dir.join("data/agent.db");

        // 检查数据库文件
        if !db_path.exists() {
            // 从备份恢复
            let latest_backup = self.find_latest_backup().await?;
            self.restore_backup(&latest_backup).await?;
        } else {
            // 尝试修复数据库
            if let Err(e) = self.verify_database(&db_path).await {
                log::error!("Database verification failed: {}", e);
                self.restore_from_backup().await?;
            }
        }

        Ok(())
    }
}
```

### 11.2 数据恢复

```rust
// src/data/recovery.rs

pub struct DataRecovery {
    db: Arc<Database>,
    backup_dir: PathBuf,
}

impl DataRecovery {
    /// 恢复会话数据
    pub async fn recover_sessions(&self) -> Result<usize, RecoveryError> {
        // 1. 找到未完成的会话
        let incomplete_sessions = self.db
            .get_incomplete_sessions()
            .await?;

        let mut recovered = 0;

        for session in incomplete_sessions {
            // 检查会话是否超时
            if session.is_expired() {
                self.db.mark_session_abandoned(&session.id).await?;
                continue;
            }

            // 尝试恢复会话
            if self.recover_session(&session).await? {
                recovered += 1;
            }
        }

        Ok(recovered)
    }

    /// 恢复任务
    pub async fn recover_tasks(&self) -> Result<usize, RecoveryError> {
        // 找到运行中的任务
        let running_tasks = self.db
            .get_running_tasks()
            .await?;

        let mut recovered = 0;

        for task in running_tasks {
            // 标记任务为失败
            self.db.mark_task_failed(&task.id, "Application crashed").await?;

            // 通知用户
            self.notify_task_failed(&task).await?;

            recovered += 1;
        }

        Ok(recovered)
    }
}
```

## 13. 故障排查

### 13.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 应用无法启动 | 缺少依赖 | 安装 WebView2 (Windows) 或 webkit2gtk (Linux) |
| Webhook 不工作 | 端口被占用 | 修改配置文件中的端口 |
| 技能执行失败 | 权限不足 | 检查权限配置 |
| 数据库损坏 | 非正常关闭 | 从备份恢复 |
| 更新失败 | 网络问题 | 检查网络连接，手动下载更新 |
